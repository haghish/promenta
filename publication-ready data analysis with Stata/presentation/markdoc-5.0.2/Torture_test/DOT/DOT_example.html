Dynamic Software Visualization

Writing software documentation and visualizing seletive information can boost active learning in students. It can also provide very useful information about structure of a software and encourages users to read the code and contribute to an open-source project. MarkDoc package supports the DOT (graph description language), and it can produce HTML documents with graphs, written within the source code or data analysis.

The graph can be written in different sections. However, MarkDoc has a separate syntax for beginning a new graph or continuing the previous graphs. to begin a new graph, use the regular syntax for wiriting documentation plus a double dollar sign and to continue using the current graph, use a single dollar sign. It's much easier to write a detailed graph in different section and update it, when there is a change in that section. Writing a detailed graph at once and within a single section makes it less interesting and makes it feel like extra work when the job is already done.
